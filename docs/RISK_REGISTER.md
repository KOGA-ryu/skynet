# Risk Register

## Risk: UI Style Overtakes Workflow Truth

Mitigation:

- every UI surface must map to stored state and explicit contracts
- shell implementation must follow panel doctrine and gate truth

## Risk: Cloud Tasks Become Broad and Expensive

Mitigation:

- use reversible trim maps
- use bounded packets only
- keep cloud output file-based and schema-bound

## Risk: Review Becomes Ceremonial

Mitigation:

- enforce gate predicates in the backend
- reflect gate failures visibly in the shell

## Risk: Metadata Attaches To Unstable Text

Mitigation:

- metadata only after approval
- graph shaping only on approved outputs

## Risk: Reject And Rework Blur Together

Mitigation:

- keep distinct queue behavior and lineage behavior
- record distinct review outcomes in persistence

## Risk: Doctrine Rewrites After Long Sessions

Mitigation:

- phase check
- milestone check
- decision-log check
- return to the operating sentence when drift appears
